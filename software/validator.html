LLM Profiles Validator

Validate your structured data against LLM Profiles

Note: This is a tool overview. For complete documentation, visit https://llmprofiles.org/

Tool Overview

The LLM Profiles Validator helps you validate your structured data implementations against our opinionated profiles. Ensure your JSON-LD markup follows best practices for both SEO and AI applications.

Key Features

  • Profile Validation: Validate against specific LLM Profiles (FAQPage, Article, Course, etc.)
  • Schema Compliance: Check JSON-LD against page.schema.json and output.schema.json
  • AEO Optimization: Verify Answer Engine Optimization best practices
  • Training Data Export: Generate training.jsonl exports for AI applications
  • CI/CD Integration: Integrate validation into your deployment pipeline

Usage Examples

Command Line Validation

# Validate JSON-LD against FAQPage profile node scripts/validate-ajv.js faqpage/v1/page.schema.json your-markup.json # Validate extracted output node scripts/validate-ajv.js faqpage/v1/output.schema.json your-extracted-data.json

CI/CD Integration

# GitHub Actions workflow - name: Validate LLM Profiles run: | node scripts/validate-json.js node scripts/validate-ajv.js faqpage/v1/page.schema.json examples/faqpage/minimal.page.jsonld

API Validation

# Fetch and validate profile const profile = await fetch('https://llmprofiles.org/faqpage/v1/index.jsonld'); const schema = await fetch('https://llmprofiles.org/faqpage/v1/page.schema.json');

Validation Rules

The validator checks for:

  • Required Fields: All mandatory fields for the profile
  • Data Types: Correct data types and formats
  • Stable IDs: Persistent @id values for entities
  • Language Hints: Proper inLanguage specification
  • Evidence Anchors: isBasedOn and url references
  • Disambiguation: sameAs links for entity resolution

Available Profiles

Validate against these profiles:

  • FAQPage - FAQ pages with Q&A pairs
  • Article - Blog posts and articles
  • Course - Educational courses
  • Event - Event information
  • ProductOffer - Product listings
  • QAPage - Single Q&A pages

Getting Started

  1. Choose a profile that matches your content type
  2. Download the validation schemas from the profile directory
  3. Implement JSON-LD markup following the profile guidelines
  4. Run validation using the provided scripts
  5. Integrate validation into your CI/CD pipeline
